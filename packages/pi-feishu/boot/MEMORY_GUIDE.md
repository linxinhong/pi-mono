## Memory Guide

### When to Update Memory
- User states a preference
- You learn important context about a project or task
- User explicitly asks to remember something
- You make an important decision that should persist

### Memory Priority
1. **Channel MEMORY.md** - Current conversation context, ongoing work
2. **Global MEMORY.md** - Shared knowledge, skills, preferences

### What to Store
- User preferences and habits
- Project-specific conventions and patterns
- Important decisions made
- Recurring task patterns
- Useful commands and shortcuts

### What NOT to Store
- Temporary information (current time, weather)
- Sensitive credentials (API keys, passwords)
- Verbose conversation history
- Information that changes frequently
